# BLIP-2 Explorer: Domain Specialized Underwater Image Captioning

**Course:** CMPT 420 — Simon Fraser University  
**Authors:** James Nguyen · Kevin Tang · Mason Liang  
**Teacher Model:** [Salesforce/blip2-opt-2.7b](https://huggingface.co/Salesforce/blip2-opt-2.7b) (~3.7B params)  
**Student Model:** [Salesforce/blip-image-captioning-large](https://huggingface.co/Salesforce/blip-image-captioning-large) (~400M params)  
**Dataset:** [UICD — Underwater Image Captioning Dataset](https://www.kaggle.com/datasets/kevintang2048/underwater-image-captioning-dataset-uicd) (3,176 images)  
**Accelerator:** Kaggle Notebooks — 2× NVIDIA T4 GPU

---

## Abstract

Marine robotics and ecological monitoring produce large volumes of underwater imagery, yet turning this footage into searchable text requires vision-language models that are both domain-aware and deployable on embedded hardware. Existing models are either too large for on-device use or too generic to handle underwater visual challenges such as color attenuation, low contrast, and domain-specific subjects.

We introduce **BLIP-2 Explorer**, a lightweight captioning pipeline that adapts BLIP-2 to underwater imagery via low-rank adaptation (LoRA) and compresses it further through knowledge distillation into a compact BLIP-1 student. Training uses a composite objective combining cross-entropy loss with a CLIP auxiliary alignment term; checkpoint selection uses a weighted composite of five caption metrics to prevent metric gaming.

## Results

Evaluated on 319 test images from UICD against all five ground-truth reference captions per image.

| Model | CLIP ↑ | CIDEr ↑ | SPICE ↑ | METEOR ↑ | ROUGE-L ↑ | BLEU-4 ↑ |
|-------|--------|---------|---------|----------|-----------|---------|
| Base BLIP-2 (frozen) | 0.311 | 0.474 | 0.138 | 0.181 | 0.331 | 0.110 |
| LoRA BLIP-2 | 0.310 | 1.159 | 0.243 | 0.307 | 0.632 | 0.403 |
| **LoRA aug. BLIP-2 (teacher)** | **0.313** | **1.317** | **0.267** | **0.333** | **0.663** | **0.439** |
| Distilled BLIP-1 (student) | 0.321 | 0.703 | 0.190 | 0.305 | 0.435 | 0.238 |

Key gains from base to LoRA augmented BLIP-2: **CIDEr +179%**, **SPICE +93%**, **BLEU-4 +300%**.  
The distilled student achieves a **4× GPU memory reduction** (5.5 GB → 1.47 GB) while retaining competitive quality.

## Approach

### Stage 1 — Teacher: BLIP-2 + LoRA
- Base `Salesforce/blip2-opt-2.7b` fully frozen; LoRA adapters (r=16, α=16, dropout=0.1) injected into `q_proj`, `k_proj`, `v_proj` of Q-Former cross-attention and OPT decoder.
- **Training objective:** Cross-entropy loss + CLIP auxiliary alignment loss (α=0.2), applied every 5 steps with linear warmup over epoch 1.
- **Checkpoint selection:** Weighted composite score — CLIP (0.50), CIDEr (0.25), METEOR (0.10), BLEU-4 (0.08), ROUGE-L (0.07). Best checkpoint at epoch 1.
- **Post-processing:** Regex stripping of degenerate trailing period sequences (e.g. `"A fish swims . . . . ."`), which would otherwise collapse CIDEr from 1.159 to 0.017.

### Stage 2 — Student: BLIP-1 + LoRA + Knowledge Distillation
- Student `Salesforce/blip-image-captioning-large` (~400M params, 9× smaller than teacher).
- Trained on **teacher pseudo-labels** (captions generated by frozen teacher over the training split) using the same LoRA config and composite loss.
- Evaluated on the same test set with original ground-truth references for direct comparison.
- Achieves **4× memory reduction** at ~53% of teacher CIDEr, with METEOR and ROUGE-L largely preserved.

### Training Details
- **Optimizer:** AdamW (lr 2×10⁻⁴, weight decay 0.01)
- **Schedule:** Linear warmup over 10% of steps
- **Batch:** Size 3 with 4-step gradient accumulation (effective batch 12)
- **Epochs:** 5, with gradient checkpointing enabled
- **Inference:** 8-bit quantization (BitsAndBytes), beam search (3 beams, max 50 tokens)

## Setup

### 1. Create the Conda environment

```bash
conda env create -f environment.yml
conda activate blip2-sentry
```

### 2. Run on Kaggle (recommended)

The notebooks are designed for **Kaggle Notebooks with 2× T4 GPUs**. Upload the notebooks and attach the UICD dataset:
- [`kevintang2048/underwater-image-captioning-dataset-uicd`](https://www.kaggle.com/datasets/kevintang2048/underwater-image-captioning-dataset-uicd)

For the distillation notebook (`blip1-explorer-distill.ipynb`), also attach the teacher pseudo-labels dataset:
- `kevintang2048/blip2-explorer-pseudo-labels`

Run notebooks in order:
1. `blip2-explorer-lora.ipynb` — train teacher
2. `blip1-explorer-pseudo.ipynb` — generate pseudo-labels
3. `blip1-explorer-distill.ipynb` — train student
